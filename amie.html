Branching

Branching allows us to develop features in isolation (away from the main branch), that we can later merge into our main(production) branch

To create a new branch - you can do it in 2 steps:

  • git branch nameOfNewBranch
  • git checkout nameOfNewBranch
  • OR in 1 command type: git checkout -b nameOfNewBranch

Merging

To merge new changes that someone else has added to the remote repository I can PULL them down

The PULL is going to bring the repository from the REMOTE (github) to our Local Repository AND merge it into our WORKING DIRECTORY where our local repo now matches the remote repo

good to note this can be done in two individual steps (FETCH and MERGE) however we will be using PULL going forwards

Pull Requests

A pull request allows other developers in your team to review the code you've written on new features, branches etc before being MERGED into the main (production) branch

It encourages an open discussion around changes to the code base with the other developers. It allows visibility and transparency around the updates/changes, before they are approved and the branches are merged
