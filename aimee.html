Thoughts about Git and Github

Things I know:

  • I can clone a repository down from Github to my local repository
  • I can add changes from my working directory to the staging area, commit them to my local repo and then push them up to Github
  • If I pull changes down from Github to my local repo, this also merges them with my working directory
  • If I want to pull changes down in two stages, I can use fetch and then merge (but I probably won't do this much)
  • I can create a new branch of a repo, so I can work on a new feature without affecting the main branch
  • When I want to merge my new feature into the main branch, I need to submit a pull request.

Things I wonder:

  • How does it work when there are lots of devs working on different branches at the same time?
